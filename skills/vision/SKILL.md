# vision — clarify the summit, top-down

> **Namespace.** Addressed as `mcs:<project>:vision` (`mcs_namespace.qualify("vision")`).
> **Paths.** Brain at `$BRAIN_ROOT` (`mcs_paths.brain_root()`); never hardcode a home path.

Where `manifest` builds UP from the most fundamental customer problems, **`vision` works
DOWN from the meta-initiatives.** The founder focuses directly on the 24-month thrusts —
making each one **more specific, sharper, more concrete** — and that clarity cascades
through the tree.

## The flow
1. **Pick the meta-initiative(s)** to sharpen (read the current meta-initiative tree + the
   genesis decomposition first, so this EXTENDS the plan rather than reinventing it).
2. **Clarify, specifically** — interrogate each with Musk's algorithm (question the
   requirement and name a real owner; "the system needs it" is not an owner). Make the
   success state concrete and falsifiable; name the real gating dependencies.
3. **Cascade** — propagate the sharpened initiative DOWN into its projects → subprojects →
   tasks; re-sequence where the clarification changes priority.
4. **Reflect onto the board** — update the initiative/project/subproject tree top-down with
   the new specificity + provenance.
5. **Feed the autonomous loop** — the newly-clarified, now-actionable work becomes units in
   the loop.

## Relationship to the trio
`vision` (top-down) and `manifest` (bottom-up) meet in the middle — the meta-initiatives get
sharper from above while the fundamental problems get solved from below. `ramble` is how the
founder pours new thinking into either. All three converge on the autonomous stretch toward
the vision.

## Owner routing (base-roster template — genesis proposes the rest)
Cascaded tasks route to the owning agent. Ship with the **base roster** only and let the
genesis roster proposer fill the company-specific additions from real evidence:
**Coordinator / chief-of-staff** (orchestration / cross-cutting) · **Specialist A / B / C**
(the named domain lanes — legal/business, product, software/build) · **operator** (anything
only the founder can gate). Route to a single domain owner when one is nameable; reserve the
coordinator for genuinely cross-cutting work.

## Rails
- Sharpening + cascading edits to the plan are reversible and local; never push unasked.
- No raw private bodies / secrets / PII in any committed file.
- The four hard limits (root `CLAUDE.md` §4) apply unchanged.
